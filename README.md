<h3><a id="user-content-agrold_etl" class="anchor" aria-hidden="true" href="#agrold_etl"><svg class="octicon octicon-link" viewBox="0 0 16 16" version="1.1" width="16" height="16" aria-hidden="true"><path fill-rule="evenodd" d="M4 9h1v1H4c-1.5 0-3-1.69-3-3.5S2.55 3 4 3h4c1.45 0 3 1.69 3 3.5 0 1.41-.91 2.72-2 3.25V8.59c.58-.45 1-1.27 1-2.09C10 5.22 8.98 4 8 4H4c-.98 0-2 1.22-2 2.5S3 9 4 9zm9-3h-1v1h1c1 0 2 1.22 2 2.5S13.98 12 13 12H9c-.98 0-2-1.22-2-2.5 0-.83.42-1.64 1-2.09V6.25c-1.09.53-2 1.84-2 3.25C6 11.31 7.55 13 9 13h4c1.45 0 3-1.69 3-3.5S14.5 6 13 6z"></path></svg></a>AgroLD_ETL</h3>
<p>Contains parsers and model used to convert data considered for AgroLD to RDF.</p>
<ul>
<li><a href="/SouthGreenPlatform/AgroLD_ETL/tree/master/model">model</a>: All documents who describe how data are transformed</li>
<li><a href="/SouthGreenPlatform/AgroLD_ETL/tree/master/rdf_ttl">rdf_ttl</a>: All output of transformation sort by dataset</li>
<li><a href="/SouthGreenPlatform/AgroLD_ETL/tree/master/riceKB">riceKB</a>: Contains scripts used for each data set</li>
<li><a href="/SouthGreenPlatform/AgroLD_ETL/tree/master/test_files">test_files</a>: All test files in input ( heterogeneous format: csv, tabbed files, gff3 )</li>
<li><a href="/SouthGreenPlatform/AgroLD_ETL/tree/master/riceKBpipeline.py">riceKBpipeline.py</a>: Script file where we have centralised all execution</li>
</ul>
<p>The type of each dataset is different, GFF, HapMap, CSV and VCF. In first time we have developed parser for build a dictonary,
because is easy to browse a dictionary and create the RDF</p>
